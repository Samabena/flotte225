# Reference: MoSCoW Prioritization

When asking the user to prioritize a feature, always explain the four options:

- **Must Have** — Cannot launch without this. Core functionality.
- **Should Have** — Important but not critical for launch. Add it soon after.
- **Could Have** — Nice to have. Only if time and budget allow.
- **Won't Have (yet)** — Explicitly out of scope for now. Future roadmap.

Always suggest a priority based on context, then ask the user to confirm:

"I'd suggest **Must Have** since users can't log in without it.
Does that sound right, or would you like a different priority?"

Never assign a priority silently — always show your reasoning and give
the user a chance to change it.
